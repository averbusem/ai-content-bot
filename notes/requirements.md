ТЕХНИЧЕСКИЙ СТЕК И ТРЕБОВАНИЯ:

1. Framework: aiogram==3.22.0
   - Использовать F-фильтры для обработки callback'ов и сообщений (from aiogram import F)
   - Все сообщения возвращать через return для корректной работы middleware
   - В каждом handler принимать state: FSMContext

2. Хранилище данных:
   - Redis для хранения FSM-состояний и данных об НКО
   - Инициализация: 
     storage = RedisStorage.from_url(settings.REDIS_URL)
     dp = Dispatcher(storage=storage)

3. Структура FSM:
   - Для каждого флоу создавать отдельный State класс
   - Названия состояний должны быть понятными и отражать текущий шаг

4. Стиль кода:
   - Комментарии только там, где логика неочевидна (минимум комментариев)
   - Комментарии на русском языке
   - Использовать async/await
   - Стараться не дублировать код

5. Keyboards:
   - Использовать ТОЛЬКО Inline клавиатуры, replay не нужны
   - Callback_data должны быть понятными и структурированными

6. Settings:
   - Использовать pydantic-settings для конфигурации
   - Переменные окружения: REDIS_URL, BOT_TOKEN
